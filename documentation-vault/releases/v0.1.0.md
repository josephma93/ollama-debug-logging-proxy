# v0.1.0

**Released:** 2026-05-06

## Summary

First stable release after canary validation, with stronger launchd convergence behavior and a fully structured documentation vault.

## Highlights

- Hardened launchd installer behavior with renamed `dev.ollama.*` service labels and safer reconciliation.
- Stable + canary Homebrew release channels driven by tag shape (`vX.Y.Z` vs `vX.Y.Z-<suffix>`).
- Documentation vault reorganized under Diataxis with backfilled ADR history and enforced hygiene rules.

## Changes

### Added

- Automated release workflow for tag classification, artifact publication, and formula generation.
- Diataxis bucket notes in `documentation-vault/` plus ADR backfill through ADR 0018.

### Changed

- LaunchAgent plist names and labels now use the `dev.ollama` namespace.
- Installer and uninstall scripts now reconcile existing launchd state more defensively.
- Contributor docs now route long-form design and process guidance through the vault.

### Fixed

- Homebrew packaging flow for stable/canary formula handling and formula placeholders.
- Script execute-bit packaging regression in prerelease artifacts.

### Removed

- None.

## Migration notes

Re-run `ollama-logging-proxy-install` after upgrading so launchd labels and staged plist names are reconciled. If you previously managed plists manually, run `ollama-logging-proxy-uninstall` first, then reinstall.
